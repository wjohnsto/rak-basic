Protocol walkthrough

A2A, ACP, and MCP in one demo

This page summarizes the three protocol surfaces around the demo app. A2A and ACP are HTTP endpoints on this FastAPI app. MCP is defined by mcp_server.py and is consumed by an MCP client as tools, not by the browser directly.

← Back to the demo UI
A2A

Agent-to-agent discovery and task RPC

Use the agent card for discovery, then send JSON-RPC messages to the root endpoint.

curl http://127.0.0.1:8000/.well-known/agent.json
curl -X POST http://127.0.0.1:8000/ \
  -H 'Content-Type: application/json' \
  -d '{
    "jsonrpc":"2.0",
    "method":"message/send",
    "params":{"message":{"role":"user","parts":[{"type":"text","text":"Hello from A2A"}]}},
    "id":"req-1"
  }'
curl -X POST http://127.0.0.1:8000/ \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"tasks/get","params":{"id":"TASK_ID"},"id":"req-2"}'
ACP

Agent discovery and runs over REST

ACP exposes an agent manifest at /agents and lets clients create runs at /runs.

curl http://127.0.0.1:8000/agents
curl -X POST http://127.0.0.1:8000/runs \
  -H 'Content-Type: application/json' \
  -d '{
    "agent_name":"rak-minimal-demo",
    "input":[{"role":"user","parts":[{"type":"text","content":"Hello from ACP"}]}]
  }'
curl http://127.0.0.1:8000/runs/RUN_ID
MCP

Tool access through an MCP client

The demo defines an MCP server object in mcp_server.py. An MCP client would connect to that tool surface separately from the FastAPI routes on port 8000. 

from redis_agent_kit.mcp import create_server

server = create_server(
    redis_url="redis://localhost:6379",
    name="rak-minimal-demo",
)

Use MCP when you want a tool-oriented integration. Use A2A or ACP when you want HTTP-based discovery and task submission through this app.
