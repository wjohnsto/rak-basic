Chat Tasks Protocol

Three protocols, one demo

This app exposes A2A, ACP, and MCP side-by-side. A2A and ACP are HTTP endpoints on the FastAPI server. MCP is defined in mcp_server.py and consumed by an MCP client as tools.

How it fits together:

External agents and clients connect through A2A or ACP over HTTP. The agent uses MCP tools internally for capabilities like Redis operations.

Agent-to-Agent

Discover the agent via its card, then send JSON-RPC messages to the root endpoint for task creation and retrieval. 

curl http://127.0.0.1:8000/.well-known/agent.json
curl -X POST http://127.0.0.1:8000/ \
  -H 'Content-Type: application/json' \
  -d '{
    "jsonrpc":"2.0",
    "method":"message/send",
    "params":{
      "message":{
        "role":"user",
        "parts":[{
          "type":"text",
          "text":"Hello from A2A"
        }]
      }
    },
    "id":"req-1"
  }'
curl -X POST http://127.0.0.1:8000/ \
  -H 'Content-Type: application/json' \
  -d '{
    "jsonrpc":"2.0",
    "method":"tasks/get",
    "params":{"id":"TASK_ID"},
    "id":"req-2"
  }'

Agent Communication

ACP exposes an agent manifest at /agents and lets clients create and poll runs via /runs. 

curl http://127.0.0.1:8000/agents
curl -X POST http://127.0.0.1:8000/runs \
  -H 'Content-Type: application/json' \
  -d '{
    "agent_name":"rak-minimal-demo",
    "input":[{
      "role":"user",
      "parts":[{
        "type":"text",
        "content":"Hello from ACP"
      }]
    }]
  }'
curl http://127.0.0.1:8000/runs/RUN_ID

Model Context Protocol

The demo defines an MCP server in mcp_server.py. An MCP client connects to the tool surface separately from the FastAPI routes. 

from redis_agent_kit.mcp import create_server

server = create_server(
    redis_url="redis://localhost:6379",
    name="rak-minimal-demo",
)

Use MCP when you want a tool-oriented integration. Use A2A or ACP when you want HTTP-based discovery and task submission.
